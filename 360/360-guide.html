方法一：多角度拍照旋轉 MULTI-ANGLE PHOTOGRAPHY SPIN

最推薦

將產品放在轉盤上，固定相機角度，每隔固定度數拍一張照片（通常 24~72 張），再用 JavaScript 串成可拖曳旋轉的互動效果。這是業界最常用的產品 360° 展示方式。

拍攝設備需求

  • 相機 — 任何可手動控制曝光的相機（手機也行，單眼更佳）
  • 電動轉盤 — 網拍轉盤 NT$500~3,000 或專業轉盤 NT$5,000~30,000
  • 三腳架 — 固定相機位置，確保每張照片高度一致
  • 燈光 — 柔光箱 ×2 或環形燈，確保均勻打光
  • 背景 — 白色/黑色背景布或純色背景紙

拍攝步驟（人工操作）

Step 1：場地佈置

白色背景紙鋪開，轉盤放中央，左右各放柔光箱（45° 角打光），相機上三腳架對準產品中心。

柔光箱 柔光箱 ╲ ┌─────────────────┐ ╱ ╲ │ │ ╱ ╲│ 白色背景紙 │╱ │ │ │ ┌─────────┐ │ │ │ 轉盤 │ │ ← 每次旋轉 5°~15° │ │ ┌─────┐ │ │ │ │ │ 產品 │ │ │ │ │ └─────┘ │ │ │ └─────────┘ │ │ │ └────────┬────────┘ │ 📷 相機 (三腳架固定)
Step 2：相機設定

手動模式（M mode），固定光圈 f/8~f/11、ISO 100、快門依測光調整。關閉自動白平衡，手動設定白平衡確保每張色溫一致。對焦後切到手動對焦（MF）鎖定。

Step 3：拍攝

方式 A — 電動轉盤：設定自動旋轉，每轉 10° 停一下拍一張 → 共 36 張
方式 B — 手動轉盤：每次手轉一個固定角度，拍一張
方式 C — 人繞著拍：產品固定，攝影師繞著走，每隔固定距離拍一張（精度較低）

建議張數：

  • 24 張 = 每 15° 一張（最低限度，旋轉有跳動感）
  • 36 張 = 每 10° 一張（推薦，流暢度與檔案大小平衡）
  • 72 張 = 每 5° 一張（絲滑流暢，但檔案多）
Step 4：後製

1. Lightroom 批次調整：白平衡統一、曝光微調、去背景
2. Photoshop 批次去背（或用 remove.bg 線上去背）
3. 統一裁切為正方形（建議 1000×1000 或 1500×1500 px）
4. 輸出為 JPG（品質 80%），檔名命名為 001.jpg ~ 036.jpg

Step 5：網頁串接

用 JS 套件把這些照片串成可拖曳旋轉的互動效果。

推薦 JS 套件

js-cloudimage-360-view（免費） 360-image-viewer SpriteSpin（jQuery） Sirv 360 Viewer（付費 SaaS） Magic 360（付費 plugin） 自己用 JS 寫（見下方範例）

自己用 JS 串接的程式碼範例

<!-- HTML --> <div id="viewer" style="width:600px;height:600px;cursor:grab;"> <img id="spin-img" src="images/001.jpg" style="width:100%;height:100%;object-fit:contain;"> </div> <script> const totalFrames = 36; // 你有幾張照片 const folder = 'images/'; // 照片資料夾 let current = 1; let dragging = false; let startX = 0; const img = document.getElementById('spin-img'); const viewer = document.getElementById('viewer'); // 預載所有圖片 for (let i = 1; i <= totalFrames; i++) { const preload = new Image(); preload.src = folder + String(i).padStart(3, '0') + '.jpg'; } viewer.addEventListener('pointerdown', (e) => { dragging = true; startX = e.clientX; viewer.style.cursor = 'grabbing'; }); window.addEventListener('pointerup', () => { dragging = false; viewer.style.cursor = 'grab'; }); window.addEventListener('pointermove', (e) => { if (!dragging) return; const delta = e.clientX - startX; if (Math.abs(delta) > 5) { // 靈敏度 current += delta > 0 ? 1 : -1; if (current > totalFrames) current = 1; if (current < 1) current = totalFrames; img.src = folder + String(current).padStart(3, '0') + '.jpg'; startX = e.clientX; } }); </script>

多排拍攝（上下也能轉）

如果想讓使用者也能上下翻轉看底部和頂部：

拍攝 5 排 × 36 張 = 180 張照片 排1：相機在上方 60° 俯視 → 拍 36 張 排2：相機在上方 30° 俯視 → 拍 36 張 排3：相機在平視 0° → 拍 36 張（主要排） 排4：相機在下方 -30° 仰視 → 拍 36 張 排5：相機在下方 -60° 仰視 → 拍 36 張 每一排都完整轉 360° 拖曳左右 → 水平旋轉（切換同排不同張） 拖曳上下 → 垂直旋轉（切換不同排同角度）

優點

  • 畫質最高（真實照片）
  • 客戶最信任（非虛擬）
  • 設備門檻低
  • 載入速度快
  • 手機相容好

缺點

  • 需拍攝時間（約 1~2 小時/產品）
  • 後製去背耗時
  • 每個新產品要重新拍
  • 多排拍攝設置較複雜
成本：
NT$500 ~ 5,000（自己拍）/ NT$3,000 ~ 15,000（外包一個產品）

方法二：3D 建模渲染 3D MODEL + WEBGL (THREE.JS)

專業級

用 3D 建模軟體建立產品模型，匯出為網頁格式（glTF/GLB），在網頁上用 Three.js 即時渲染。使用者可真正自由旋轉任意角度，還能加入動態光效。

建模方式

  • 手動建模 — 用 Blender（免費）/ 3ds Max / SolidWorks 建模
  • 3D 掃描 — 用 3D 掃描器掃描實體產品，生成模型
  • 照片建模（Photogrammetry） — 拍 50~200 張不同角度照片，用軟體重建 3D
  • AI 生成 — 用 Meshy / Tripo3D / CSM 等 AI 從照片生成 3D 模型

照片建模（Photogrammetry）拍攝方法

拍攝要求

1. 拍攝 50~200 張照片，涵蓋所有角度
2. 相鄰照片重疊率 > 60%
3. 環繞產品拍 3 圈：平視、俯視 45°、俯視 75°
4. 光線均勻，避免強反光（金屬產品要噴霧消光）
5. 背景簡單（不要太雜亂）

俯視 75° 圈 ──→ ○ ○ ○ ○ ○ ○ ○ ○ ○ （約 15~20 張） ╲ ╱ 俯視 45° 圈 ──→ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ （約 20~30 張） ╲ ╱ 平視 0° 圈 ──→ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ （約 25~40 張） │ ┌───────┐ │ 產 品 │ └───────┘ │ 仰視圈（如可）──→ ○ ○ ○ ○ ○ ○ ○ ○ ○ （底部細節）

照片建模軟體

Meshroom（免費開源） Agisoft Metashape（專業付費） RealityCapture（最快速） Polycam（手機 App） KIRI Engine（手機 App） Luma AI（雲端 AI）

網頁呈現流程

3D 模型（.blend / .obj / .fbx） │ ▼ 匯出為 glTF / GLB 格式（Blender 直接匯出） │ ▼ 壓縮優化（用 gltf-transform 或 glTF Pipeline） 目標：< 5MB 才適合網頁 │ ▼ Three.js + GLTFLoader 載入 加入 OrbitControls 讓使用者旋轉 │ ▼ 部署到網站

Three.js 載入範例

import * as THREE from 'three'; import { GLTFLoader } from 'three/addons/loaders/GLTFLoader.js'; import { OrbitControls } from 'three/addons/controls/OrbitControls.js'; const loader = new GLTFLoader(); loader.load('fishing-light.glb', (gltf) => { scene.add(gltf.scene); }); // 使用者可自由旋轉、縮放 const controls = new OrbitControls(camera, renderer.domElement); controls.enableDamping = true;

優點

  • 真正任意角度旋轉
  • 可加動態光效/發光
  • 一個模型多種用途
  • 可展示內部結構（爆炸圖）

缺點

  • 建模技術門檻高
  • 外包費用高
  • 金屬質感難還原
  • 手機效能可能不足
成本：
NT$10,000 ~ 80,000（外包建模+網頁）/ 免費（自學 Blender）

方法三：360° 環景影片/照片 360° VIDEO / PANORAMA

適合場景展示

用 360° 全景相機拍攝，讓使用者在網頁上可以自由轉動視角觀看。注意：這是「環境的 360°」（觀看者站在中心向外看），不是「產品的 360°」（觀看者繞著產品看）。適合展示產品使用場景，不適合展示產品細節。

拍攝設備

  • Insta360 X4 / X3 — NT$12,000~18,000，拍照錄影都行
  • Ricoh Theta Z1 — NT$28,000，畫質極好的 360 相機
  • GoPro MAX — NT$15,000，防水耐操
  • 手機全景模式 — 免費但品質有限（只能拍平面全景）

拍攝場景建議（漁業用燈）

場景 A：甲板夜間照明

360° 相機放在漁船甲板中央，LED 燈開啟，展示夜間照明範圍。可以讓觀看者感受身歷其境在船上的感覺。

場景 B：水面集魚效果

360° 相機架在船舷邊，拍攝 LED 燈垂入水中的實況，展示水面下的光照效果和魚群聚集情況。

場景 C：安裝現場

在港口或船上展示 LED 燈的安裝位置、佈線方式、多盞燈的排列效果。

網頁呈現方式

Pannellum（免費開源） A-Frame（免費，WebXR） Photo Sphere Viewer（免費） Google VR View Kuula（SaaS 平台） Momento360（免費上傳）

Pannellum 使用範例

<!-- 引入 Pannellum --> <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/pannellum/build/pannellum.css"> <script src="https://cdn.jsdelivr.net/npm/pannellum/build/pannellum.js"></script> <div id="panorama" style="width:100%;height:500px;"></div> <script> pannellum.viewer('panorama', { type: 'equirectangular', panorama: 'fishing-boat-360.jpg', // 你的 360° 照片 autoLoad: true, autoRotate: -2, compass: true, hotSpots: [{ pitch: -10, yaw: 120, type: 'info', text: 'LED 漁業用燈安裝位置' }] }); </script>

優點

  • 臨場感極強
  • 展示使用場景最佳
  • 拍攝簡單（一鍵拍）
  • 可加熱點說明

缺點

  • 不能仔細看產品細節
  • 需購買 360° 相機
  • 影片檔案很大
  • 夜間場景畫質可能差
成本：
NT$12,000 ~ 28,000（相機）+ 免費（軟體）

方法四：CSS/JS 3D 模擬 PROGRAMMATIC 3D SIMULATION

零素材

不需要任何照片或 3D 模型，用程式碼（Three.js 幾何體）直接建構產品的 3D 示意模型。適合當 prototype / demo 使用，就像你目前的 index.html。

適合的情境

  • 產品還沒做好，但需要先給客戶看概念
  • 沒有拍攝條件，先用模擬展示功能特點
  • 需要展示光效、散熱、結構等抽象概念
  • Demo / 提案 / 概念驗證階段

技術方案

Three.js（當前使用） Babylon.js CSS 3D Transform p5.js / WebGL

優點

  • 完全免費、零素材
  • 馬上可以做出來
  • 可互動（開關燈、換色）
  • 不需要拍照或建模

缺點

  • 不是真實產品外觀
  • 只能當概念展示
  • 細節有限
  • 正式場合略顯不足
🔗

方法五：混合方案（推薦正式使用） HYBRID APPROACH

正式商用推薦

實務上最好的做法是組合多種方式，在同一個產品頁面上提供完整體驗：

建議組合

┌─────────────────────────────────────────────────┐ │ 產品頁面 │ │ │ │ ┌───────────────────────────────────────────┐ │ │ │ 區塊 1：多角度拍照旋轉 │ │ │ │ → 36 張照片 360° 轉盤 │ │ │ │ → 讓客戶看清楚產品外觀、質感、做工 │ │ │ └───────────────────────────────────────────┘ │ │ │ │ ┌───────────────────────────────────────────┐ │ │ │ 區塊 2：3D 互動展示（選配） │ │ │ │ → Three.js 模型，可開關燈、換色 │ │ │ │ → 展示功能特點（透視、爆炸圖） │ │ │ └───────────────────────────────────────────┘ │ │ │ │ ┌───────────────────────────────────────────┐ │ │ │ 區塊 3：360° 場景體驗 │ │ │ │ → 360° 全景照片/影片 │ │ │ │ → 在漁船上的實際使用情況 │ │ │ └───────────────────────────────────────────┘ │ │ │ │ ┌───────────────────────────────────────────┐ │ │ │ 區塊 4：產品規格 + 比較表 + CTA │ │ │ └───────────────────────────────────────────┘ │ └─────────────────────────────────────────────────┘

為什麼混合最好

  • 多角度照片 → 真實感和信任感
  • 3D 互動 → 展示功能亮點
  • 360 場景 → 臨場感和情境
  • 涵蓋所有客戶需求

考量

  • 製作時間較長
  • 需要多種素材
  • 網頁較重（需做好 lazy loading）
  • 維護成本較高

今天 Demo 用
